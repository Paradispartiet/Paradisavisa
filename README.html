🌑 Paradisavisa

Paradisavisa er en ideologisk og samfunnskritisk nettavis bygget som et statisk nettsted på GitHub Pages.
Forsiden (index.html) genererer automatisk nyheter, sport, kultur, kommentarer og debatt fra flere JSON-filer.
Systemet er lettvekts, raskt, og fullt vedlikeholdbart direkte via GitHub – uten behov for CMS.

Dette er en videreutvikling av den opprinnelige versjonen av Paradisavisa, som brukte én samlet posts.json-fil. I dag er innholdet delt opp i flere seksjoner – et “hjul”-system – som gir bedre oversikt og dynamikk.

📰 Legge til ny artikkel

  1. Lag en ny fil i /artikler/, f.eks.: 
    artikler/2025-10-21-paris-bilfri.html
  2. Bruk en mal fra Maler/ (for eksempel mal-nyhet.html eller mal-kultur.html).
  3. Legg artikkelen inn i riktig JSON-fil: 
    {
  "title": "Paris kutter 500 gater for biler",
  "excerpt": "Fra motorvei til menneskeby – byen fullfører sitt urbane eksperiment.",
  "image": "Nyhetshjul/Paris.PNG",
  "url": "artikler/2025-10-21-paris-bilfri.html",
  "date": "2025-10-21"
}

Tips

  • Husk komma , mellom objektene i JSON-filene.
  • Nyeste artikler vises automatisk øverst (sortert på date).
  • Datoformat: YYYY-MM-DD.

🔄 Versjonshåndtering og cache

Paradisavisa bruker to nivåer av versjonsstyring for å sikre at endringer lastes riktig i nettleseren:

NivåBrukes tilNår du skal endre
?v= i HTMLTvinger nettleseren til å hente ny CSS/JSVed små designendringer
const VERSION i service-worker.jsFull cache-resetVed nye filer eller strukturendringer

📋 Anbefalt rutine

  1. Endre ?v= når du oppdaterer design eller små detaljer.
  2. Endre VERSION i service-worker.js når du legger til nye filer.
  3. Etter commit → vent 1–2 minutter → oppdater siden med Ctrl + F5.

⚙️ Publisering via GitHub Pages

  1. Last opp eller rediger filer direkte i GitHub.
  2. Trykk Commit changes.
  3. GitHub Pages oppdaterer automatisk (tar 1–2 min).
  4. Sjekk nettstedet:
    https://<brukernavn>.github.io/Paradisavisa/

Hvis oppdateringen ikke vises:

  • Øk ?v=-nummeret i index.html.
  • Eller oppdater VERSION i service-worker.js.
  • Oppdater siden med Ctrl + F5.

🧠 Vedlikeholdssammendrag

  • Data ligger i JSON-filer i rotmappen.
  • Forsiden oppdateres automatisk når JSON endres.
  • Artikler legges i /artikler/ og lenkes fra riktig JSON.
  • GitHub Pages hoster alt statisk – ingen server nødvendig.
  • Bruk ?v= for små endringer, VERSION for større.
  • Gamle filer (posts.json) beholdes for referanse.
